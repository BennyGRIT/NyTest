Förklaringar på termer #3:

Working Directory

I working directory gör du alla ändringar, som att lägga till nya filer (märks som untracked), ändra i filer (märks som modified) och ta bort filer (märks som deleted). Genom att använda git status kan du få aktuell status för ditt working directory, och se om det finns ändringar som behöver läggas till staging area.


Staging

Efter att ändringar gjorts i "working directory" markerar detta kommando ändringar som gjorts som staged, innebärande att de är redo för en "commit". Med kommandot git status kan du kontrollera om det finns ändringar som är "staged" eller inte.

HEAD

Git sparar alla ändringarna i en tidslinje som det går att spola framåt och bakåt i. Tillståndet som ett repository för tillfället står på kallas för HEAD (som i “läshuvud”).

Revert

Både kommandona git revertoch git resetkommandona ångrar tidigare åtaganden. Men om du redan har tryckt på ditt åtagande till ett fjärrförvar rekommenderas det att du inte använder det git reseteftersom det skriver om historien om åtaganden. Detta kan göra det svårt att arbeta på ett arkiv med andra utvecklare och upprätthålla en konsekvent historia om åtaganden. Istället är det bättre att använda git revert, vilket ångrar ändringarna som gjordes av ett tidigare åtagande genom att skapa ett helt nytt engagemang, allt utan att ändra historiken för åtaganden.


Reset

Med git resetkommandot kan du återställa ditt nuvarande huvud till ett angivet tillstånd. Du kan återställa tillståndet för specifika filer såväl som en hel gren. Det här är användbart om du inte har gjort ditt engagemang till GitHub eller ett annat fjärrförvar än.
